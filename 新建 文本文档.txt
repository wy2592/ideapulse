IdeaPulse v0.1 - 最终版开发需求文档 (PRD)

目标受众: AI 代码生成工具 (如 Cursor, Copilot, Claude Code)

核心约束: 无数据库、全内存、极简、防炸、安全。

1. 项目概述 (What is IdeaPulse?)

IdeaPulse 是一个零注册、零追踪的实时点子验证工具。

它像一个“创意对讲机”：用户发送一句话点子，瞬间广播给所有在线陌生人，对方通过红绿滑块打分，结果以“警灯”颜色（红/黄/绿）实时反馈给发送者。

核心价值: 在 30 秒内告诉你“这个点子该做还是该放弃”。

2. 核心架构原则 (Core Principles)

No Database (严禁数据库):

绝对禁止使用 SQL / NoSQL / 文件存储。

所有状态（用户配额、队列、活跃点子）仅存在于 Node.js 内存​ 中。服务重启即清零。

Hard Resource Limits (硬资源限制):

并发处理池大小固定（50 免费 + 10 付费）。

等待队列长度固定（最大 1000）。

连接数固定（最大 8000）。

Volatile State (易失状态):

前端历史（三盏灯）仅存于浏览器内存，关闭窗口即消失，不写 LocalStorage。

Fail Open (熔断优先):

AI 预审超时或报错时，默认放行点子，不阻塞队列。

4. 功能规格 (Feature Specs)
4.1 前端 (Preact + Tauri)

主窗口 (320x80px):

左侧：三盏灯（初始灰色空心）。

右侧：在线人数 + "Send Idea" 按钮。

发送流程:

点击按钮 → 弹出 Modal（仅含一个 Textarea + Launch 按钮）。

输入 ≤200 字符，点击 Launch → Modal 关闭。

对应的一盏灯变为 黄色呼吸（Pending）。

接收流程:

收到 NEW_IDEA→ 显示点子文本 + 红绿渐变滑块。

拖动滑块松手 → 发送 VOTE。

结果反馈 (核心视觉):

收到 RESULT→ 对应灯变为 红/黄/绿，并触发 CSS 动画（见下文）。

4.2 后端 (Node.js + ws)

连接管理:

生成 deviceId(UUID)。

维护 usersMap: { deviceId, dailyQuota, votesToday, isPaid }。

配额系统:

免费：基础 1 + 投票 10 次 +1 + 投票 30 次 +1 = 3次/日。

付费：10次/日。

处理池 (Admit Pool):

实现两个独立的 Semaphore/Pool（公共 50，Pro 10）。

只有拿到 Slot 才能调用 AI。

优先级队列:

使用最小堆 (Min-Heap)。

优先级：Pro(0)> DeepUser(1)> ActiveUser(2)> Normal(3)。

AI 预审:

调用 OpenAI Moderation API。

熔断: setTimeout(3000)，超时则 resolve(true)。

8. 开发交付清单 (Implementation Checklist)

请按此顺序生成代码：

Backend (/backend)

[ ] server.js: WS 服务器，连接限制，Ping/Pong 心跳。

[ ] pool.js: Semaphore​ 实现（Admit Pool）。

[ ] queue.js: Priority Queue​ 实现（最大 1000，支持淘汰）。

[ ] gatekeeper.js: AI 调用逻辑（含 3s 超时熔断）。

[ ] quota.js: 内存配额管理，每日重置逻辑。

Frontend (/frontend)

[ ] App.jsx: 主组件，三灯状态管理。

[ ] Modal.jsx: 发送点子的弹窗。

[ ] Slider.jsx: 红绿渐变滑块（Canvas 或 CSS 实现）。

[ ] styles.css: 包含所有灯效动画。

Config

[ ] .env.example: PORT, OPENAI_KEY, SKIP_AI=false, POOL_SIZE=50.

9. 特别注意事项 (Critical Warnings)

严禁使用 Express 渲染页面，只用 ws。

严禁使用 localStorage​ 存储任何点子内容或历史。

必须清理: 每个 activeIdeas里的 setTimeout必须在结算或用户断线时 clearTimeout，防止内存泄漏。

Pro Key: 初期用简单的 Set内存存储，不要对接支付系统。
